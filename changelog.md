# Журнал изменений

## Версия 0.5.0 (Не опубликовано)

### Улучшения структуры и интерфейса
- **Полная переработка системы меню**: Внедрена новая архитектура динамического меню, адаптирующегося к роли пользователя и контексту
- **Шаблон "Фабрика"**: Создан сервис `KeyboardFactory` для централизованного создания клавиатур различных типов
- **Контекстная навигация**: Реализована система контекстов меню с сохранением состояния и возможностью возврата к предыдущему экрану
- **Модульность интерфейса**: Меню теперь разделено на логические разделы (профиль, заявки, пользователи, админка)

### Новый функционал
- **Улучшенная система репутации**: Добавлены обработчики для голосования за пользователей с указанием причин
- **Расширенное управление заявками**: Реализованы callback-обработчики для различных действий с заявками
- **Административная панель**: Добавлены инструменты для управления пользователями, заявками и статистикой
- **Система навигации**: Создана функция `handleNavigation` для управления перемещением между разделами бота

### Улучшения кода
- **Типобезопасность**: Усиление типизации в сервисах клавиатур и обработчиках контекста
- **Разделение ответственности**: Четкое разделение между сервисами создания клавиатур и их использованием
- **Унифицированный интерфейс**: Единый подход к созданию меню во всех контекстах приложения
- **Расширяемость**: Структура кода упрощает добавление новых разделов и функций меню

### Технические улучшения
- **Хранение состояния**: Добавлены поля `menuContext` и `previousContext` в сессионные данные
- **Обработка ошибок**: Реализована функция `safeAnswerCallback` для надежной обработки callback-запросов
- **Проверки прав доступа**: Каждый обработчик проверяет наличие необходимых прав у пользователя
- **Код-стиль**: Улучшена читаемость кода и добавлены подробные комментарии к методам

## Версия 0.4.0 (2023-06-20)

### Улучшения структуры
- **Реорганизация кода**: Выделен отдельный файл `bot.ts` для инициализации бота и `routes.ts` для маршрутизации
- **Типы данных**: Вынесены типы и интерфейсы в отдельные файлы в директории `models`
- **Разделение ответственности**: Прямые обработчики перенесены из `index.ts` в соответствующие контроллеры
- **Обработка ошибок**: Реализована централизованная система обработки ошибок в `utils/errorHandler.ts`
- **Сбор метрик**: Добавлен `metricsMiddleware` для отслеживания производительности запросов

### Исправления ошибок
- **Исправлен баг с правами доступа**: Удалено условие `ctx.message.text?.startsWith('/')` из `adminMiddleware`, которое неправильно позволяло всем командам обходить проверку прав администратора
- **Доработаны тесты**: Исправлены импорты в тестовых файлах для корректной работы с Bun
- **Регистрация контроллеров**: Подключены все контроллеры, включая ранее пропущенный `ratingController`

### Улучшения кода
- **Утилитарные функции**: Добавлены вспомогательные функции для проверок в middleware
- **Константы**: Вынесены строковые константы для предотвращения "магических строк"
- **Типизация**: Усилена типизация в сервисах и контроллерах
- **Middleware**: Улучшена структура middleware и порядок их применения

### Дополнительные изменения
- **Документация**: Добавлены JSDoc комментарии к ключевым функциям
- **Порядок инициализации**: Исправлен порядок инициализации сервисов и компонентов
- **Оптимизация импортов**: Устранены потенциальные циклические зависимости
